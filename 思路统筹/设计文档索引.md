## 四、Phase 2 安全与稳定性

### 4.1 沙箱架构

| 组件 | 选择 | 理由 |
|------|------|------|
| 代码执行沙箱 | Pyodide | WebAssembly，部署简单 |
| 状态持久化 | Redis CheckpointSaver | 断点续跑支持 |
| 错误分类 | 5级分类 | 可重试 vs 不可重试 |
| 日志 | 结构化 JSON | task/node 上下文 |

### 4.2 错误处理流程

```
错误发生
    │
    ▼
分类错误 ──▶ 是否可重试？
    │
    ├── 是 ──▶ 指数退避 ──▶ 重试 ──▶ 成功？
    │                        │
    │                        ├── 是 ──▶ 继续
    │                        └── 否 ──▶ 最大重试次数？
    │                                    │
    │                                    ├── 是 ──▶ 标记失败
    │                                    └── 否 ──▶ 继续重试
    │
    └── 否 ──▶ 直接标记失败
```

## 五、Phase 3 评测体系

### 5.1 测试集规模

| 类别 | 用例数 | 说明 |
|------|--------|------|
| 检索类 (RET) | 10-20条 | 特定方法文献、公式出处 |
| Text-to-Code (CODE) | 10-20条 | 指标构造、回归分析、预测建模 |
| 引用溯源 (CITE) | 10-20条 | 方法综述、结果分析段落 |
| 端到端 (E2E) | 10-15条 | 完整流程 |

### 5.2 核心指标

| 指标 | 阈值 |
|------|------|
| RAG Recall@5 | > 0.9 |
| RAG Precision@10 | > 0.7 |
| Text-to-Code 执行成功率 | > 0.85 |
| 引用溯源 Trace Success Rate | > 0.9 |
| 引用漂移 Drift Rate | < 0.05 |
| 端到端任务完成率 | > 0.7 |

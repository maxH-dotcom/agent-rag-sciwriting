## 一、任务创建到首次中断流程

### 1.1 完整首次中断时序（normal flow）

```mermaid
sequenceDiagram
    actor U as 用户
    participant FE as 前端工作台
    participant API as FastAPI
    participant LG as LangGraph主控
    participant DM as data_mapping
    participant LIT as literature
    participant NOV as novelty

    rect rgb(230, 245, 255)
        Note over U,DM: 阶段1：用户创建任务
        U->>FE: 上传数据文件 + 输入研究问题
        FE->>API: POST /tasks<br/>{user_query, data_files, paper_files}
        API->>LG: create_initial_state()<br/>→ run_until_pause()
        LG-->>API: task created, status="running"
        API-->>FE: TaskResponse<br/>{task_id, status="running"}
    end

    rect rgb(240, 255, 240)
        Note over LG,DM: 节点1：data_mapping（自动执行）
        LG->>DM: data_mapping_node.run(state)
        DM->>DM: 解析CSV列名
        DM->>DM: 检测数据类型
        DM->>DM: 检测面板结构
        DM->>DM: 推荐变量映射
        DM-->>LG: state.data_mapping_result<br/>status="interrupted"<br/>interrupt_reason="data_mapping_required"
    end

    rect rgb(255, 250, 240)
        Note over LG,LIT: 节点2：literature（自动继续）
        LG->>LIT: literature_node.run(state)
        LIT->>LIT: paper-qa检索文献
        LIT->>LIT: 提取method_metadata
        LIT-->>LG: state.literature_result<br/>status="interrupted"<br/>interrupt_reason="literature_review_required"
    end

    rect rgb(255, 240, 255)
        Note over LG,NOV: 节点3：novelty（自动继续）
        LG->>NOV: novelty_node.run(state)
        NOV->>NOV: 迁移可行性评估
        NOV->>NOV: 生成transfer_assessments
        NOV->>NOV: 推荐研究方向
        NOV-->>LG: state.novelty_result<br/>status="interrupted"<br/>interrupt_reason="novelty_result_ready"
    end

    rect rgb(245, 245, 245)
        Note over LG,API: LangGraph暂停，返回最终状态
        LG-->>API: final_state<br/>status="interrupted"<br/>interrupt_reason="novelty_result_ready"
        API-->>FE: TaskResponse<br/>{task_id, status="interrupted",<br/>interrupt_reason, interrupt_data}
        FE-->>U: 展示novelty中断面板<br/>（迁移评估结果 + 决策按钮）
    end
```

### 1.2 首次中断触发点详解

```mermaid
sequenceDiagram
    participant U as 用户
    participant System as 系统

    rect rgb(220, 235, 250)
        Note over System: 执行到第一个有意义的决策点
        System->>System: data_mapping完成 → 中断1触发
        System->>System: literature完成 → 中断2触发
        System->>System: novelty完成 → 中断3触发（首次有意义的用户决策点）
    end

    Note over U,System: 首次中断策略：在novelty节点暂停<br/>原因：用户需要确认研究方向后再继续分析
```

## 二、用户确认后恢复执行流程

### 2.1 用户确认后继续（approved flow）

```mermaid
sequenceDiagram
    actor U as 用户
    participant FE as 前端工作台
    participant API as FastAPI
    participant LG as LangGraph主控
    participant ANA as analysis
    participant BRI as brief
    participant WRI as writing

    Note over U,FE: 用户已收到novelty_result_ready中断
    Note over U,FE: 已审核迁移评估结果

    rect rgb(230, 255, 230)
        Note over U,API: 用户点击"确认方向"（approved）
        U->>FE: 点击"确认选题方向"
        FE->>API: POST /tasks/{id}/continue<br/>{decision: "approved"}
        API->>LG: from_task_response(task)<br/>→ run_until_pause(resume=True)
    end

    rect rgb(255, 255, 240)
        Note over LG,ANA: 从novelty恢复，继续执行analysis
        LG->>ANA: analysis_node.run(state)<br/>current_node="analysis"<br/>human_decision={decision:"approved"}
        ANA->>ANA: 基于novelty_result制定分析计划
        ANA->>ANA: 调用Text-to-Code Bridge
        ANA->>ANA: 生成代码并执行
        ANA-->>LG: state.analysis_result<br/>status="interrupted"<br/>interrupt_reason="code_plan_ready"
    end

    rect rgb(255, 250, 240)
        Note over LG,BRI: 自动继续到brief
        LG->>BRI: brief_builder_node.run(state)
        BRI->>BRI: 汇总所有节点结果
        BRI->>BRI: 组装Research_Brief
        BRI-->>LG: state.brief_result<br/>status="interrupted"<br/>interrupt_reason="brief_ready_for_review"
    end

    rect rgb(255, 240, 255)
        Note over LG,WRI: 自动继续到writing
        LG->>WRI: writing_node.run(state)
        WRI->>WRI: 读取Research_Brief
        WRI->>WRI: 生成论文提纲
        WRI->>WRI: 生成方法/结果草稿
        WRI-->>LG: state.writing_result<br/>status="interrupted"<br/>interrupt_reason="draft_ready"
    end

    rect rgb(245, 245, 245)
        Note over LG,FE: 系统在writing节点暂停
        LG-->>API: final_state<br/>status="interrupted"<br/>interrupt_reason="draft_ready"
        API-->>FE: TaskResponse<br/>{task_id, status="interrupted",<br/>interrupt_reason, interrupt_data}
        FE-->>U: 展示draft_ready中断面板<br/>（草稿预览 + 确认/修改/重新生成按钮）
    end
```

## 三、用户修改决策后的恢复流程

### 3.1 用户修改代码后继续（modified flow - analysis阶段）

```mermaid
sequenceDiagram
    actor U as 用户
    participant FE as 前端工作台
    participant API as FastAPI
    participant LG as LangGraph主控
    participant ANA as analysis
    participant BRI as brief
    participant WRI as writing

    Note over U,FE: 用户已收到code_plan_ready中断
    Note over U,FE: 查看了生成的代码，发现需要修改

    rect rgb(230, 245, 255)
        Note over U,API: 用户点击"修改代码"并提交修改
        U->>FE: 点击"修改代码"
        FE->>U: 展示代码编辑器
        U->>FE: 修改代码内容
        FE->>API: POST /tasks/{id}/continue<br/>{decision: "modified",<br/>payload: {code_script: "修改后的代码"}}
        API->>LG: apply_human_payload(state, payload)<br/>→ merge payload into analysis_result
    end

    rect rgb(255, 255, 240)
        Note over LG,ANA: 修改后重新执行analysis
        LG->>ANA: analysis_node.run(state)<br/>current_node="analysis"<br/>human_decision={decision:"modified",<br/>payload:{code_script:"..."}}
        Note right of ANA: 检测到human_decision.type="modified"<br/>跳过代码生成<br/>直接用修改后的code_script<br/>重新执行沙箱
        ANA->>ANA: 使用修改后的code_script
        ANA->>ANA: 重新执行沙箱
        ANA-->>LG: state.analysis_result (更新)<br/>status="running"
    end

    rect rgb(240, 255, 240)
        Note over LG,BRI: 继续到brief
        LG->>BRI: brief_builder_node.run(state)
        BRI-->>LG: state.brief_result<br/>status="interrupted"<br/>interrupt_reason="brief_ready_for_review"
    end

    LG-->>API: final_state (interrupted at brief)
    API-->>FE: TaskResponse<br/>status="interrupted"<br/>interrupt_reason="brief_ready_for_review"
    FE-->>U: 展示brief中断面板
```

## 四、用户拒绝/中止流程

### 4.1 用户在novelty阶段拒绝（rejected flow - novelty）

```mermaid
sequenceDiagram
    actor U as 用户
    participant FE as 前端工作台
    participant API as FastAPI
    participant LG as LangGraph主控

    Note over U,FE: 用户已收到novelty_result_ready中断
    Note over U,FE: 检查了所有研究方向，均不满意

    rect rgb(255, 230, 230)
        Note over U,FE: 用户点击"拒绝"
        U->>FE: 点击"拒绝"（rejected）
        FE->>API: POST /tasks/{id}/continue<br/>{decision: "rejected"}
        API->>LG: continue_task(task_id, human_decision)
    end

    rect rgb(255, 240, 240)
        Note over LG,LG: LangGraph处理拒绝
        LG->>LG: _prepare_resume()<br/>decision="rejected"
        LG->>LG: status="aborted"<br/>interrupt_reason="user_rejected_current_stage"<br/>error_message="用户拒绝了novelty阶段"
    end

    rect rgb(245, 245, 245)
        Note over LG,FE: 立即返回，不继续执行
        LG-->>API: final_state<br/>status="aborted"
        API-->>FE: TaskResponse<br/>{task_id, status="aborted"}
        FE-->>U: 任务已终止<br/>展示终止状态页面
    end
```

### 4.2 用户在任意阶段拒绝（通用拒绝流程）

```mermaid
sequenceDiagram

    LG-->>API: {task_id, status="aborted", abort_reason}
    API-->>FE: TaskResponse
    FE-->>U: 任务已终止<br/>reason: {abort_reason}
```

### 4.3 系统错误导致的终止（error flow）

```mermaid
sequenceDiagram
    actor U as 用户
    participant FE as 前端工作台
    participant API as FastAPI
    participant LG as LangGraph主控
    participant ANA as analysis

        ANA->>ANA: 代码执行失败
        ANA-->>LG: state<br/>status="error"<br/>error_message="沙箱执行超时"

    rect rgb(255, 240, 240)
        Note over LG,LG: 错误处理
        LG->>LG: 错误分类<br/>error_type="retryable"

            LG->>LG: 指数退避重试

            LG->>LG: status="error"<br/>不重试
        end
    end

    opt 重试耗尽
        LG-->>API: final_state<br/>status="error"<br/>error_message
        API-->>FE: TaskResponse
        FE-->>U: 执行失败<br/>error: {error_message}<br/>可重试 / 需人工修复
    end
```

## 五、中断点详细时序

### 5.1 六个中断点触发时序对照表

| 中断点 | 触发时机 | 中断后展示 | 用户可做决策 |
|--------|----------|------------|--------------|
| 中断1 | data_mapping完成 | 变量映射建议 | 确认/修改/取消上传 |
| 中断2 | literature完成 | 文献列表+方法元信息 | 确认/增删改/跳过 |
| 中断3 | novelty完成 | 迁移评估+推荐方向 | 确认/修改/拒绝 |
| 中断4 | analysis完成 | 代码+执行结果 | 确认/修改/跳过 |
| 中断5 | brief完成 | Research Brief预览 | 确认/直接修改JSON |
| 中断6 | writing完成 | 论文草稿预览 | 确认/修改/重新生成 |

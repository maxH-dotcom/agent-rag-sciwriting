# Bad Case 记录规范

该目录用于沉淀真实数据验收、前后端联调、任务中断流和结果展示中的失败样例。

规则：

1. 任何一次真实数据测试失败，都必须在本目录新增记录文件。
2. 不允许只在对话、终端日志或临时笔记中保留失败信息。
3. 一个失败样例对应一个独立文件，推荐命名：
   - `YYYY-MM-DD-数据文件名-问题编号-失败节点.md`
4. 修复后不得删除原记录，应在原记录中补充“修复方案”和“回归结果”。

每条 bad case 记录至少包含以下字段：

```md
# 标题

- 日期：
- 数据文件：
- 问题编号：
- 用户问题：
- 失败节点：
- 相关任务 ID：

## 复现步骤
1.
2.
3.

## 预期结果

## 实际结果

## 报错信息

## 根因分析

## 解决方案

## 修复后回归结果

## 备注
```

建议：
- 若涉及前端报错，附上页面路径和组件名
- 若涉及后端报错，附上 API 路由、节点名和关键日志
- 若涉及真实数据问题，明确是 CSV 还是 Excel，以及具体字段名

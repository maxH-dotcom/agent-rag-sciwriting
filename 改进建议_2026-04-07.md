# 智能科研工作助手 - 用户体验改进建议

> 基于 2026-04-07 首次完整使用体验
> 用户场景：预测浙江省未来5年碳排放

---

## 一、核心体验总结

### 1.1 优点

- 多Agent编排，流程完整（数据映射→文献→创新性→代码→Brief）
- 人工中断点设计合理，可控性强
- API文档清晰，可通过API完整操作
- 面板数据解析能力较强

### 1.2 痛点

- **意图理解偏差**：用户说"时间序列预测"，系统生成了"面板回归"
- **配置分散**：Zotero、OpenAlex等API配置在README中，无界面
- **沙箱不完整**：执行代码失败，缺少依赖
- **模型类型单一**：主要是面板计量模型，缺少时间序列、机器学习模型

---

## 二、具体改进建议

### 2.1 研究问题解析增强

**问题**：用户输入 `method_preference: 时间序列`，系统未正确响应

**建议**：
- 在问题解析阶段，增加**方法类型识别**（面板回归 vs 时间序列 vs 机器学习 vs 深度学习）
- 对时间序列场景，自动识别需要：历史数据聚合（省级总量）、平稳性检验、ARIMA/SARIMA/Prophet/LSTM等模型池
- 在数据映射阶段，提示用户确认：`您选择的是 [时间序列预测]，我们将对11个城市数据聚合为省级时间序列后建模`

**优先级**：高

---

### 2.2 统一配置中心

**问题**：Zotero API、OpenAlex API、数据路径等配置分散

**建议**：
- 新增 `/settings` 页面，包含：
  - API配置区（Zotero API Key、OpenAlex Token）
  - 路径配置区（默认数据路径、默认文献路径）
  - 沙箱配置区（Python环境路径、依赖列表）
  - 模型偏好区（默认模型类型、LLM选择）
- 配置持久化到 `~/.research_assistant/config.json`

**优先级**：高

---

### 2.3 模型推荐池扩展

**问题**：当前只支持面板计量模型（固定效应、随机效应、STIRPAT）

**建议**：根据研究问题类型，自动扩展模型池

| 问题类型 | 推荐模型池 |
|---------|-----------|
| 时间序列预测 | ARIMA、SARIMA、Prophet、XGBoost时间序列、LSTM |
| 面板数据分析 | 固定效应、随机效应、差分GMM、系统GMM |
| 机器学习预测 | XGBoost、LightGBM、Random Forest + SHAP解释 |
| 深度学习预测 | LSTM、GRU、Transformer |
| 因果推断 | 双重差分DID、合成控制法SCM、断点回归RDD |

**优先级**：高

---

### 2.4 沙箱环境修复

**问题**：代码执行失败 `ModuleNotFoundError: pandas`

**建议**：
- 方案A：沙箱启动时自动安装 `requirements.txt` 中的依赖
- 方案B：提供"在本地环境运行"按钮，导出代码 + 数据打包下载
- 方案C：沙箱使用与后端相同的 `.venv` 环境

**优先级**：高

---

### 2.5 数据预处理自动化

**问题**：年份列是字符串类型，导致PanelOLS报错

**建议**：
- 增加**数据类型自动检测**：数字字符串 → 自动转换数值型
- 增加**缺失值报告**：自动检测并报告各列缺失情况
- 增加**异常值提示**：自动检测极值并提示用户确认

**优先级**：中

---

### 2.6 预测结果可视化

**问题**：当前只输出回归系数，无预测图表

**建议**：
- 时间序列任务：自动生成预测折线图（含置信区间）
- 面板回归任务：生成各城市碳排放趋势对比图
- 驱动因素任务：生成系数重要性条形图 + SHAP解释图

**优先级**：中

---

### 2.7 论文文件智能解析

**问题**：文献检索质量依赖Zotero或手动上传

**建议**：
- 支持拖拽上传PDF到聊天界面
- 自动解析PDF元数据（标题、作者、年份、摘要）
- 内置碳排放领域关键词规则，自动匹配相关论文段落
- 支持从DOI/URL直接导入论文

**优先级**：中

---

### 2.8 Research Brief模板优化

**问题**：当前Brief主要是代码和配置的堆砌

**建议**：生成结构化论文草稿：
```
## 1. 研究背景
## 2. 数据与方法
   - 2.1 数据描述（表格）
   - 2.2 模型设定
   - 2.3 变量定义
## 3. 实证结果
   - 3.1 描述性统计
   - 3.2 回归结果
## 4. 结论与政策建议
## 5. 参考文献
```

**优先级**：低

---

### 2.9 任务历史与复用

**问题**：每次都要重新输入研究问题和数据路径

**建议**：
- 保存历史任务模板
- 支持"基于上次任务修改"快速创建
- 支持导出完整任务配置为JSON

**优先级**：低

---

### 2.10 中断点交互优化

**问题**：5个中断点略显繁琐，高级用户可能希望减少确认

**建议**：
- 增加"专家模式"：跳过部分中断点
- 中断点显示"预计耗时"和"产出说明"
- 支持拖拽调整变量映射（可视化界面）

**优先级**：低

## 四、技术实现注意事项

1. **意图识别**：建议用LLM在问题解析阶段做方法类型分类prompt，避免硬编码规则
2. **配置中心**：建议用 `pydantic-settings` 管理配置，支持环境变量覆盖
3. **模型池**：建议抽象 `ModelBase` 类，每类模型实现 `fit`, `predict`, `summary` 接口
4. **沙箱**：建议用 `venv` 或 `conda` 环境隔离，不要用空白沙箱

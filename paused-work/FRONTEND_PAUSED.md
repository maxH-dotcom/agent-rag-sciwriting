# 前端暂停说明

## 当前位置

前端相关代码已从仓库主线目录移出，统一放在：

- `paused-work/frontend-workbench-paused/`

这样做是为了把当前开发重心锁定在后端，避免前后端一起改导致目录结构和任务优先级再次发散。

## 为什么暂停

当前产品最关键的问题不在页面，而在后端主链路还没有完全接上真实能力：

1. 文献检索仍然是 fallback 适配层
2. 状态持久化还没有切到 LangGraph + Redis
3. 中断后的“修改并继续”刚完成后端基础支持，还需要先稳定接口契约

这时候继续堆前端，容易出现一种很典型的问题：
页面越做越像产品，后端其实还是假链路。看着热闹，不耐用。

## 当前前端里有什么

这个暂停目录里保留了：

- Next.js 基础配置
- 工作台首页
- 任务创建表单
- 中断查看与继续/终止按钮
- 任务列表
- 任务详情页

它们目前更像“联调用壳子”，不是正式交付的 UI。

## 恢复前端开发前，建议先完成的后端事项

1. 真实 `paper-qa` / OpenAlex 文献检索适配
2. LangGraph + Redis 持久化
3. API 层的任务历史、继续、终止、错误分支测试
4. 文件上传和路径校验策略

## 恢复时建议

恢复前端时，不要直接在这个目录里继续散着改。

建议先确认：

- API 契约已经稳定
- 中断点字段不会频繁变
- 后端测试已覆盖主链路

然后再决定是：

1. 把该目录移回主线
2. 或者重建一个更正式的 `frontend/`，只迁移保留的页面逻辑

## 结论

前端不是删除了，是被“收进仓库侧边柜”了。

现在先把后端做实，后面恢复前端会更快，也更稳。
